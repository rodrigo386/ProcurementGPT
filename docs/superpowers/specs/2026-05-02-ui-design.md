## 1. Contexto

A camada Chat (sub-projeto 4) entregou `POST /api/chat` em Edge Runtime, transmitindo via Vercel AI SDK (`streamText` + `StreamData`). Faltava a interface web para usuários conversarem com o bot.

Este sub-projeto entrega `app/(chat)/page.tsx`: uma SPA cliente (split-pane, sidebar de conversas em localStorage + área de chat) montando `useChat` do AI SDK contra `/api/chat`. Também faz um retrofit em `lib/rag/prompt-builder.ts` para remover citações `[N]` da resposta do modelo (decisão de produto: usuário não vê fontes).

O critério de pronto é: `npm run dev`, abrir `/chat`, clicar uma sugestão, mensagem é enviada, resposta streama renderizada como markdown, sessão é salva em localStorage e aparece na sidebar; refresh recarrega; trocar de sessão funciona.

### 5.1 `lib/chat-storage.ts` (pure)

```ts
export type StoredSession = {
  id: string;
  title: string;
  messages: ChatMessage[];   // imported from @/lib/rag/types
  updatedAt: number;
};

const KEY = 'pgpt:sessions:v1';
const CAP = 50;

export function loadSessions(): StoredSession[] { /* try/catch → [] */ }
export function saveSessions(sessions: StoredSession[]): void { /* cap to 50, sort updatedAt desc */ }
export function createSession(): StoredSession { /* crypto.randomUUID, [], 'Nova conversa', Date.now() */ }
export function deriveTitle(messages: ChatMessage[]): string { /* first user, trim 60 + ellipsis, fallback 'Nova conversa' */ }
```

All functions wrap localStorage in try/catch — SSR, private mode, quota exceeded all return safe defaults / log `console.warn`. No throws.

### 5.2 `hooks/useChatSessions.ts`

```ts
export function useChatSessions(): {
  sessions: StoredSession[];
  currentId: string;
  current: StoredSession;
  switchTo(id: string): void;
  createNew(): void;
  deleteSession(id: string): void;
  updateMessages(messages: ChatMessage[]): void;  // also updates title from first user msg
};
```

Reads `loadSessions()` on mount inside `useEffect` (avoids SSR mismatch). If empty, calls `createNew()` automatically. `updateMessages` derives title and writes to localStorage.

### 5.3 `components/chat/ChatRoot.tsx` and `ChatSession.tsx`

**Two components**, split because `useChat`'s `initialMessages` is read once on mount. Switching sessions requires remounting `useChat` — handled by `key={currentId}` on a child component.

`ChatRoot.tsx` (parent, owns sessions):
```tsx
'use client';
export function ChatRoot() {
  const { sessions, currentId, current, switchTo, createNew, deleteSession, updateMessages } = useChatSessions();
  return (
    <div className="flex h-screen bg-background text-foreground">
      <Sidebar sessions={sessions} currentId={currentId} onSwitch={switchTo} onNew={createNew} onDelete={deleteSession} />
      <div className="flex-1 flex flex-col min-w-0">
        <Header />
        <ChatSession key={currentId} session={current} onMessagesChange={updateMessages} />
      </div>
    </div>
  );
}
```

`ChatSession.tsx` (child, owns the AI SDK hook for the active session):
```tsx
'use client';
export function ChatSession({ session, onMessagesChange }: { session: StoredSession; onMessagesChange: (m: ChatMessage[]) => void }) {
  const { messages, input, setInput, handleSubmit, isLoading, stop } = useChat({
    api: '/api/chat',
    id: session.id,
    initialMessages: session.messages,
    onFinish: (msg) => onMessagesChange([...messages, msg]),
  });
  return (
    <>
      {messages.length === 0
        ? <EmptyState onPick={(text) => setInput(text)} />
        : <MessageList messages={messages} isLoading={isLoading} />}
      <Composer input={input} onChange={setInput} onSubmit={handleSubmit} isLoading={isLoading} onStop={stop} />
    </>
  );
}
```

The `key={currentId}` causes React to unmount/remount `ChatSession` when the user switches sessions — guaranteeing `useChat` re-initializes with the fresh `initialMessages`. This is the documented React pattern for "reset child state when prop changes" and avoids any need for `setMessages` plumbing.

`app/(chat)/page.tsx` simply renders `<ChatRoot />` (no other logic — keeps the route file dumb).

## 10. Decisões e justificativas

| Decisão | Por quê |
|---|---|
| Split-pane sidebar with localStorage history (Q1-B) | User pick. Sub-projeto 6 will replace localStorage with DB; keeping the contract `useChatSessions` lets us swap implementations without touching components. |
| Electric Blue accent (Q2) | User pick. Single CSS variable means easy reskin when brand lands. |
| No user-facing citations (Q3) | User pick. Cleaner UX; trust comes from response quality, not visible chips. Retrieval still grounds. |
| Full markdown rendering (Q4) | User pick. Model produces structured 4-part responses; raw text would look like a debugger. |
| Hero + 4 suggestion cards (Q5) | User pick. Maps to classifier intents; doubles as user education + onboarding smoke test. |
| All defaults (Q6) | User pick. System theme, drawer for mobile, Enter sends, +Nova in sidebar, stop button live, header minimal. |
| Route group `(chat)` | Lets `/` stay as the existing landing while `/chat` is the app. Zero file conflict. |
| No e2e for v1 | Manual smoke is faster to maintain at this stage. Sub-projeto 7 can add Playwright when the test surface stabilizes. |
| Component tests via per-file `@vitest-environment jsdom` | Avoids restructuring `vitest.config.ts` into projects. Smaller surface change, easier to revert. |
| Sources annotation kept in stream but unrendered | Forward-compat: future admin view can read it without re-plumbing. Cost: zero — it's already there. |

## 11. Riscos

| Risco | Mitigação |
|---|---|
| `useChat` v4 instance-per-id semantics — switching sidebar entries might leave stale state | The `id` prop on `useChat` documented to swap state. Smoke step 5 (refresh + switch) catches regressions. |
| localStorage quota at 50 sessions × long messages | Cap + FIFO eviction; saves wrapped in try/catch. Dev tools warn on quota exceed. |
| react-markdown XSS surface | `react-markdown` defaults to safe (no raw HTML). We don't enable `rehype-raw`. |
| jsdom + AI SDK in component tests | We don't test `useChat` directly — mock messages props. Avoids the need for jsdom + fetch + SSE. |
| shadcn CLI version drift might pick up unintended new components | Add only the listed primitives; review the diff before committing. |
| `prose dark:prose-invert` colors clash with brand | Inspect rendered Markdown in dark mode during smoke; tweak via `prose-headings:text-foreground` if needed. |
| Mobile drawer + stop button + safe-area-inset-bottom on iOS | shadcn Sheet handles backdrop and ESC; safe-area handled by Tailwind's `pb-[env(safe-area-inset-bottom)]`. |
